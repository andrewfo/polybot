USER PROMPT — what the model actually reads

You are a superforecaster analyzing a prediction market.

{date_context_line}  // "Today is 2026-05-14. Market resolves 2026-06-30, 47 days."

Market question: "{question}"  // "Will BTC reach $120,000 by June 30?"
Market category: {category}  // "crypto"
Current market price: {market_price}  // 0.35
Market resolution date: {end_date}

Signal analysis from multiple sources:
- Source: resolution_crypto (DIRECT RESOLUTION SOURCE)
  Estimate: 0.42
  Confidence: 0.85
  Reasoning: "BTC at $104,200. Terminal model: 38%, Barrier model: 46%..."
  Data points analyzed: 30
  [Raw evidence block: current price, target, volatility, models]

- Source: web_search
  Estimate: 0.40
  Confidence: 0.55  (already discounted 0.7×)
  Reasoning: "Recent ETF inflows bullish but macro headwinds..."
  Data points analyzed: 5
  [key_evidence: "BlackRock adds $2B...", "Fed signals pause..."]

- Source: prediction_markets
  Estimate: 0.37
  Confidence: 0.60
  Reasoning: "Kalshi: 0.33, Metaculus: 0.40, consensus 0.37"

- Source: onchain_flow
  Estimate: 0.44
  Confidence: 0.50
  Reasoning: "Net exchange outflow -12,400 BTC, whale accumulation..."

Preliminary weighted estimate: {0.41}  // pre-computed from signal weights
Signal agreement: {mixed}

Instructions:
1. Critically evaluate each signal source...
2. Signals marked "DIRECT RESOLUTION SOURCE" → weight heavily
3. Check resolution criteria vs. data source mismatch
3b. Evaluate barrier vs terminal model applicability
4. Consider base rates
5. Consider what the market knows that signals don't
6. Final probability estimate
7. Confidence rating (0-1)
8. Reasoning in 2-3 sentences

Respond as JSON only:
{
  "final_probability": 0.XX,
  "confidence": 0.XX,
  "reasoning": "...",
  "signals_agreement": "agree"|"mixed"|"disagree",
  "market_efficiency_assessment": "underpriced"|"overpriced"|"fair"
}
▼

Output → Kelly Criterion → Trade Decision

If |final_prob - market_price| > edge threshold → size with Kelly → execute

Frontier Aggregation — THE Trade Decision

signals/aggregator.py:519 · Frontier tier · Task: estimate_probability · NO FALLBACK

Input Data (assembled from all signals)

  • System prompt Date context + urgency tier + calibration guidance (from temporal.py)
  • question Market question
  • category Market category
  • market_price Current implied probability from Polymarket
  • end_date Resolution date
  • signals_block All 4 signal sources formatted with estimates, confidence, reasoning, and raw evidence
  • preliminary_prob Pre-computed weighted average from signal weights
  • signal_agreement Pre-computed agreement level (agree/mixed/disagree)

Key Instructions in Prompt

  • Resolution sources "DIRECT RESOLUTION SOURCE" signals weighted more heavily than news
  • Source mismatch Check if resolution criteria uses different exchange/methodology than signal data
  • Barrier vs Terminal Evaluate which model type applies, override if wrong
  • Anti-anchoring "Do not anchor to market price — form your own estimate from evidence"
  • On failure ALERT AND SKIP — never falls back to cheap model

Expected Output

{ "final_probability": 0.41, "confidence": 0.72, "reasoning": "CoinGecko barrier model at 46% aligns with on-chain accumulation signals. Web search confirms bullish ETF flows but macro uncertainty caps upside.", "signals_agreement": "mixed", "market_efficiency_assessment": "underpriced" }
